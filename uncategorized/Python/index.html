面向对象

6.1 面向对象简介

​ 我们前面学习的程序，都是面向过程的程序。而这一节，我们将走近面向对象编程的技术。这一节在Python基础中对初学者来说，是唯一一个相对难理解，比较抽象的内容，希望大家能多多练习。

6.1.1 面向过程与面向对象

​ 对于面向过程和面向对象的解释，我整篇文章都会通过一个小例子来解释。

​ 假设，你刚刚创办了一个公司。但是现在你的公司只有你一人，那么你自己就得负责运营，负责技术，公司的活几乎是你一个人干完，这就是面向过程。

​ 而通过这个例子来理解面向过程的优缺点，公司是你一人的，所以你很清楚公司里的每个事情，这就说明了，面向过程的代码复杂度不高。但是，因为你公司只有你一人，干活效率慢，大工程你是干不了的。而且有时候你干活的时候，可能弄错了某样东西，那么你可能又得重头开始干。这就说明了面向过程在干大型的项目的时候的一个短板，以及在出现错误时，很可能会影响整个程序。

​ 现在，你的公司做大了。你招了很多人，然后你把负责运营的弄成一个部门，负责技术的弄成一个部门。而你就负责和客户谈生意，然后把谈到的单子给下面的各个部门干。这就是面向对象。

​ 而现在，因为有各个部门的分工合作，你的干活效率大大提高，哪个部门出现错误，就让那个部门负责处理。这就是面向对象的优点，使程序的扩展性更强，适合大型的项目以及在出现错误时及时修改而不影响程序。但是，面向对象也是有缺点的。假如你的公司做的更大了，难免会有人出现如贪污等问题，还有你能很清楚公司里面的每一件事吗，这就考虑你对公司的控制力如何了。这就是面向对象的缺点，可控性较差。
